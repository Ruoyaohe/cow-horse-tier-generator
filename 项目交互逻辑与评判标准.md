# 牛马等级生成器 - 交互逻辑与评判标准文档

## 一、项目概述

牛马等级生成器是一个基于工作数据量化评估的趣味性应用，通过输入工作相关数据，系统会自动计算并生成对应的"牛马等级"鉴定结果。

## 二、交互逻辑

### 2.1 用户输入流程

#### 输入字段
1. **日均工时** (dailyHours)
   - 类型：数字输入
   - 范围：1-24 小时
   - 单位：H（小时）

2. **月休天数** (leaveDays)
   - 类型：数字输入
   - 范围：0-31 天
   - 单位：D（天）

3. **连续工作天数** (continuousDays)
   - 类型：数字输入
   - 范围：0-365 天
   - 单位：D（天）

4. **单餐伙食标准** (mealAllowance)
   - 类型：数字输入
   - 范围：0-999 元
   - 单位：￥（元）

5. **月薪区间** (salaryRange)
   - 类型：下拉选择
   - 选项：
     - 3k以下
     - 3k-6k
     - 6k-10k
     - 10k-15k
     - 15k-20k
     - 20k以上

### 2.2 交互流程

```
用户输入数据 → 点击"生成鉴定结果"按钮
    ↓
卡片翻转动画（3D翻转效果）
    ↓
显示加载动画（随机5-8秒）
    ↓
显示随机加载提示信息（每2秒切换）
    ↓
计算评分并生成结果
    ↓
根据等级随机选择对应图片
    ↓
显示鉴定结果卡片
    ↓
用户可选择：
  - 保存卡片（生成PNG图片）
  - 我要吐槽（匿名发送到小群）
  - 重新鉴定（返回输入界面）
  - 查看群聊（查看所有吐槽内容）
  - 显示二维码（移动端预览）
```

系统在加载过程中会随机显示以下提示信息（每2秒切换一次）：

1. "正在量化你的怨气值..."

3. "正在评估周一综合症程度..."
4. "正在分析年假被折现的可能性..."
5. "正在识别眼神中的清澈与愚蠢..."
6. "正在解析PPT中的无效信息..."
7. "正在同步食堂饭菜的油腻程度..."
8. "正在扫描你与工位的高度契合感..."
9. "正在雕刻你的灵魂（牛马版）..."

### 2.4 卡片保存功能

- **保存格式**：PNG图片
- **图片质量**：3倍像素比，高质量渲染
- **图片尺寸**：动态计算，确保内容完整显示
- **图片内容**：
  - 鉴定等级和标签
  - 等级徽章（带渐变效果）
  - 1:1正方形图片
  - 描述文字
  - 数据统计（日均、连续、口粮）
  - 水印和时间戳

### 2.5 二维码预览功能

- 点击右下角二维码按钮
- 显示访问二维码
- 支持移动端扫描预览
- 自动检测局域网IP地址
- 支持手动输入IP地址

### 2.6 吐槽功能

- **触发位置**：鉴定结果页面
- **功能描述**：用户可以匿名分享自己的"牛马"故事
- **输入限制**：最多500字
- **保存方式**：匿名发送到小群（保存到服务器）
- **反馈提示**："吐槽已发送小群"
- **数据保存**：吐槽内容、鉴定等级、得分、工作数据、时间戳

### 2.7 群聊功能

- **触发位置**：首页左下角群聊按钮
- **功能描述**：以聊天消息形式展示所有用户的吐槽内容
- **显示内容**：
  - 用户标识（牛马#1, 牛马#2...）
  - 鉴定等级标签（金牌/银牌/铜牌）
  - 得分徽章
  - 吐槽时间
  - 完整吐槽内容
  - 工作数据标签
- **排序方式**：按时间倒序（最新在前）
- **交互方式**：只读模式，无输入交互

## 三、评判标准

### 3.1 评分系统

系统采用**累加评分制**，根据以下维度计算总分：

#### 3.1.1 日均工时评分

| 工时范围 | 得分 |
|---------|------|
| > 12小时 | +10分 |
| > 10小时 | +6分 |
| > 8小时 | +3分 |
| ≤ 8小时 | +0分 |

#### 3.1.2 月薪区间评分

| 月薪区间 | 得分 |
|---------|------|
| 3k以下 | +10分 |
| 3k-6k | +7分 |
| 6k-10k | +5分 |
| 10k-15k | +3分 |
| 15k-20k | +1分 |
| 20k以上 | +0分 |

#### 3.1.3 连续工作天数评分

| 连续天数 | 得分 |
|---------|------|
| > 14天 | +10分 |
| > 7天 | +5分 |
| > 5天 | +2分 |
| ≤ 5天 | +0分 |

#### 3.1.4 月休天数评分

| 月休天数 | 得分 |
|---------|------|
| 0天 | +8分 |
| < 2天 | +4分 |
| ≥ 2天 | +0分 |

#### 3.1.5 单餐伙食标准评分

| 伙食标准 | 得分 |
|---------|------|
| < 15元 | +5分 |
| < 30元 | +2分 |
| ≥ 30元 | +0分 |

### 3.2 等级划分

根据总分将用户划分为三个等级：

#### 3.2.1 顶级金牌牛马 (GOLD)

- **条件**：总分 ≥ 25分

#### 3.2.2 资深银牌牛马 (SILVER)

- **条件**：12分 ≤ 总分 < 25分
- **标签**：资深银牌牛马
- **描述**："游走在崩溃边缘，你是办公室的中坚力量（耗材）。"
- **颜色**：蓝色 (#2563eb)
- **图片来源**：`/public/yinpai/` 文件夹
- **图片数量**：6张（随机选择 1.png - 6.png）

#### 3.2.3 潜力铜牌牛马 (BRONZE)

- **条件**：总分 < 12分
- **标签**：潜力铜牌牛马
- **描述**："尚存一线生机，偶尔还能看到下班的太阳。"
- **颜色**：橙红色 (#9a3412)
- **图片来源**：`/public/tongpai/` 文件夹

## 四、技术实现

### 4.1 核心技术栈

- **前端框架**：React 19.2.3
- **动画库**：Framer Motion 11.11.11
- **构建工具**：Vite 6.2.0
- **语言**：TypeScript 5.8.2
- **后端框架**：Express.js
- **图片处理**：Canvas API（自定义绘制）
- **二维码生成**：qrcode
- **并发运行**：concurrently

### 4.2 关键功能实现

#### 4.2.1 3D卡片翻转
- 使用 CSS 3D Transform
- Framer Motion 动画控制
- 双面卡片设计（正面：输入表单，背面：结果显示）

#### 4.2.2 图片随机选择
- 根据等级从对应文件夹随机选择
- 使用 `Math.random()` 生成随机索引
- 支持多图片文件夹（金牌6张，银牌6张，铜牌1张）

#### 4.2.3 卡片保存功能
- 使用 Canvas API 绘制完整卡片
- 3倍像素比确保高清输出
- 支持移动端和桌面端
- 自动处理图片跨域问题（转换为base64）

#### 4.2.4 二维码预览
- 自动检测局域网IP地址
- 使用 WebRTC 获取本地IP
- 支持手动输入IP地址
- 实时生成二维码

#### 4.2.5 后端服务器
- 使用 Express.js 搭建 RESTful API
- 支持 CORS 跨域请求
- 吐槽内容保存到服务器文件系统
- API 端点：
  - `GET /api/health` - 健康检查
  - `POST /api/feedback` - 保存吐槽
  - `GET /api/feedback/list` - 获取吐槽列表
  - `GET /api/feedback/:filename` - 获取单个吐槽内容

#### 4.2.6 移动端优化
- 响应式布局设计
- 移动端按钮尺寸和位置优化
- 输入框清空优化（清空时不显示0）
- 容器底部间距调整，避免按钮与卡片交叠
- 顶部对齐布局，提升移动端体验

## 七、评分逻辑总结

### 7.1 评分权重

各项指标的权重分配（最高分）：

1. **日均工时**：10分（最高）
2. **连续工作天数**：10分（最高）
3. **月薪区间**：10分（最高）
4. **月休天数**：8分
5. **伙食标准**：5分

**总分范围**：0-43分

### 7.2 等级阈值

- **金牌**：≥ 25分（约占总分58%以上）
- **银牌**：12-24分（约占总分28%-56%）
- **铜牌**：< 12分（约占总分28%以下）

### 7.3 评分特点

1. **多维度评估**：综合考虑工时、薪资、休息、连续工作、伙食等多个维度
2. **累加机制**：各项得分累加，总分越高等级越高
3. **阈值划分**：使用固定阈值划分等级，确保结果一致性
4. **随机性**：同等级结果会随机显示不同图片，增加趣味性

## 八、API接口文档

### 8.1 健康检查

**端点**：`GET /api/health`

**响应示例**：
```json
{
  "success": true,
  "message": "服务器运行正常",
  "timestamp": "2026-01-09T15:00:00.000Z"
}
```

### 8.2 保存吐槽

**端点**：`POST /api/feedback`

**请求体**：
```json
{
  "timestamp": "2026/01/09 15:00:00",
  "tier": "顶级金牌牛马",
  "score": 30,
  "profile": {
    "dailyHours": 12,
    "leaveDays": 0,
    "continuousDays": 15,
    "mealAllowance": 10,
    "salaryRange": "3k-6k"
  },
  "feedback": "用户输入的吐槽内容"
}
```

**响应示例**：
```json
{
  "success": true,
  "message": "吐槽已保存",
  "filename": "tucao-2026-01-09-15-00-00.txt"
}
```

### 8.3 获取吐槽列表

**端点**：`GET /api/feedback/list`

**响应示例**：
```json
{
  "success": true,
  "files": [
    {
      "filename": "tucao-2026-01-09-15-00-00.txt",
      "size": 512,
      "created": "2026-01-09T15:00:00.000Z",
      "modified": "2026-01-09T15:00:00.000Z"
    }
  ]
}
```

### 8.4 获取单个吐槽内容

**端点**：`GET /api/feedback/:filename`

**响应示例**：
```json
{
  "success": true,
  "content": "【牛马吐槽 - 2026/01/09 15:00:00】\n等级：顶级金牌牛马 (得分: 30)\n..."
}
```

## 九、移动端优化说明

### 9.1 布局优化

- **顶部对齐**：移动端使用顶部对齐布局，避免内容偏下
- **底部间距**：增加底部内边距（`pb-24`），为浮动按钮留出空间
- **按钮位置**：移动端按钮更靠近边缘（`bottom-4`），桌面端保持原位置
- **按钮尺寸**：移动端按钮更小（`w-12 h-12`），提升可用性

### 9.2 输入优化

- **清空处理**：清空输入框时不显示0，保持空白状态
- **响应式间距**：移动端减少卡片内边距和元素间距
- **触摸优化**：增加输入框内边距，提升移动端点击体验

### 9.3 响应式设计

- 使用 Tailwind CSS 响应式类（`md:` 前缀）
- 移动端和桌面端采用不同的布局策略
- 确保所有功能在移动端正常使用
